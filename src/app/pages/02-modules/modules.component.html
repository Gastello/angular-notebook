Angular Modules (Модулі в Angular)

У Angular модулі потрібні для організації коду та керування залежностями. Вони допомагають структурувати проєкт, роблячи його гнучким та масштабованим.

Що таке модуль у Angular?

Модуль – це контейнер для компонентів, сервісів, директив, пайпів та інших сутностей. Основний модуль будь-якого Angular-застосунку – AppModule (app.module.ts).

Ось як виглядає стандартний модуль:

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

  @NgModule({
    declarations: [ // Тут реєструємо компоненти, директиви та пайпи
    AppComponent
  ],
  imports: [ // Тут імпортуємо інші модулі
    BrowserModule
  ],
  providers: [], // Тут реєструємо сервіси
  bootstrap: [AppComponent] // Вказуємо, який компонент запускає застосунок
  })

export class AppModule { }

Основні властивості @NgModule

  • declarations – сюди додаємо компоненти, директиви та пайпи, які належать цьому модулю.

    Щоб Angular знав про ці компоненти, вони мають бути тут оголошені.

    Компоненти в одному модулі бачать лише ті компоненти, які теж оголошені в цьому ж модулі (або в імпортованому модулі).

  • imports – сюди додаємо інші модулі, які нам потрібні в цьому модулі.

    Наприклад, якщо ти хочеш використовувати ngIf чи ngFor, тобі потрібен CommonModule.

    Якщо у тебе є окремий UserModule, його треба імпортувати в AppModule, щоб працювали його компоненти.

  • providers – тут реєструємо сервіси та інші глобальні залежності.

    Якщо додати сервіс у providers, він буде доступний для всіх компонентів у цьому модулі.

    Якщо написати providedIn: 'root' у сервісі – його не потрібно додавати в providers, Angular сам зробить його глобальним.

  • bootstrap – використовується тільки в кореневому модулі (зазвичай AppModule) і визначає основний компонент, який буде відрендерений першим при запуску застосунку.

    У звичайних модулях (не AppModule) bootstrap не використовується.

  • exports - якщо ти хочеш використовувати компонент, директиву або пайп з одного модуля в іншому, ти повинен додати його туди.

    exports працює тільки з деклараціями (declarations). Ти не можеш експортувати сервіси або інші модулі.

Для того, щоб все працювало - не забудь підключити модуль в app.component.ts/app.module.ts
